InfoSphere X Data Use

InfoSphere uses X data only to help the signed-in user understand the public followed-account environment behind their X app time.

Use Case

With user authorization, InfoSphere connects the user's public X identity and summarizes public metadata from accounts the user follows. The purpose is to classify the user's followed information environment at an aggregate level.

Requested Scopes

InfoSphere requests tweet.read, users.read, and follows.read. The tweet.read scope is requested because X requires it for the user and following endpoints used by the prototype; InfoSphere does not call Tweet/post endpoints.

Endpoints Used

InfoSphere uses GET /2/users/me for public identity and GET /2/users/:id/following for followed-account public metadata.

Stored Signals

InfoSphere stores aggregate summaries only: followed accounts scanned, verified-account counts, verified-type mix, identity-verified counts, protected-account counts, broad role hints, broad country hints, reviewed ownership-family hints, news/civic profile-location mix, news/civic reviewed ownership hints, scan status, and storage-boundary text.

Data Not Stored

InfoSphere does not store Tweets, posts, timelines, feeds, DMs, likes, authored content, access tokens as dashboard data, or the raw following list.

Sharing

X-derived data is not sold, brokered, used for advertising, or shared with third parties by this prototype.
